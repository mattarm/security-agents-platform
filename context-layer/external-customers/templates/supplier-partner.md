# Security Service Template: Supplier / Partner

## What We Deliver

Standard security posture for third-party suppliers and partners who integrate with our platform via API, data feeds, or shared infrastructure.

## Security Controls

- **API Security**: OAuth 2.0 authentication required, rate limiting enforced, input validation on all endpoints
- **Data Protection**: TLS 1.2+ in transit, data encrypted at rest, PII access restricted to minimum necessary
- **Access Control**: Dedicated API credentials per partner, scoped to specific endpoints/data
- **Monitoring**: API usage monitoring, anomaly detection on access patterns, failed auth alerting

## Monitoring Level

- **Tier**: Standard
- API access logs monitored for anomalies
- CrowdStrike coverage on shared infrastructure
- Quarterly access review of partner credentials

## Compliance & Reporting

- SOC 2 Type II report available on request (annually)
- Data processing addendum provided at onboarding
- Annual third-party risk assessment questionnaire

## Incident Response

- **SLA**: Acknowledgment within 4 hours for partner-affecting incidents
- **Communication**: Designated partner security contact notified via email
- **Escalation**: Through partner account manager

## Pen Testing

- Partners may request pen test results summary (redacted) annually
- Partner-initiated pen testing requires 30 days notice and scoping approval
